Policy-as-Code Platform

A scalable and flexible authorization system using Open Policy Agent (OPA) to externalize access control rules as code.

Go to Dashboard View API Docs

Key Features

Policy-as-Code

Define authorization rules in Rego language. Policies are versioned, testable, and can be updated without application redeployment.

Attribute-Based Access Control

ABAC model supports user roles, departments, time-based rules, and resource attributes for fine-grained access control.

Centralized Policy Engine

Open Policy Agent (OPA) serves as a dedicated decision engine, separating authorization logic from application code.

JWT Authentication

Secure token-based authentication with bcrypt password hashing. User attributes embedded in JWT for policy evaluation.

Audit Logging

Complete audit trail of all authorization decisions for compliance, debugging, and security analysis.

Dynamic Policy Updates

Update, enable, or disable policies at runtime without restarting the application. Changes take effect immediately.

Quick Start

# 1. Install dependencies
pip install -r requirements.txt

# 2. Initialize the database with sample data
python init_db.py

# 3. Start the Flask application
python run.py

# 4. (Optional) Start OPA server for full functionality
opa run --server opa_policies/

# 5. Access the dashboard
# Open http://localhost:5000 in your browser
