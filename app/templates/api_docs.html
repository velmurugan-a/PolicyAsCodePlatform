API Documentation

Complete reference for all REST API endpoints

Base URL

http://localhost:5000

Authentication

All protected endpoints require a JWT token in the Authorization header:

Authorization: Bearer <your_jwt_token>

Authentication Endpoints

POST /auth/register

Register a new user account

# Request
curl -X POST http://localhost:5000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "newuser",
    "email": "newuser@example.com",
    "password": "password123",
    "role": "employee",
    "department": "engineering"
  }'

# Response
{
  "status": "success",
  "message": "User registered successfully",
  "data": {
    "user": {
      "id": 1,
      "username": "newuser",
      "email": "newuser@example.com",
      "role": "employee",
      "department": "engineering"
    }
  }
}
POST /auth/login

Authenticate and get JWT token

# Request
curl -X POST http://localhost:5000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "admin123"
  }'

# Response
{
  "status": "success",
  "message": "Login successful",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "admin",
      "department": "management"
    }
  }
}
GET /auth/profile Auth Required

Get current user profile

# Request
curl http://localhost:5000/auth/profile \
  -H "Authorization: Bearer YOUR_TOKEN"

Policy Endpoints

POST /policy/evaluate Auth Required

Evaluate an authorization request against OPA policies

# Request
curl -X POST http://localhost:5000/policy/evaluate \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "read",
    "resource": {
      "type": "document",
      "department": "engineering"
    }
  }'

# Response (Allowed)
{
  "status": "success",
  "data": {
    "decision": "allow",
    "allowed": true,
    "reason": "Admin has full access to all resources",
    "input": {
      "user": {"role": "admin", "department": "management"},
      "action": "read",
      "resource": {"type": "document", "department": "engineering"}
    }
  }
}

# Response (Denied)
{
  "status": "success",
  "data": {
    "decision": "deny",
    "allowed": false,
    "reason": "Employee cannot access other department data"
  }
}
GET /policy/list Auth Required

List all policies

# Request
curl http://localhost:5000/policy/list \
  -H "Authorization: Bearer YOUR_TOKEN"

# Optional: Filter by status
curl "http://localhost:5000/policy/list?status=active" \
  -H "Authorization: Bearer YOUR_TOKEN"
POST /policy/create Admin Only

Create a new Rego policy

# Request
curl -X POST http://localhost:5000/policy/create \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "custom_policy",
    "description": "Custom access control policy",
    "version": "1.0.0",
    "policy_code": "package authz\ndefault allow := false\nallow { input.user.role == \"admin\" }",
    "is_active": true
  }'
PUT /policy/update/<id> Admin Only

Update an existing policy

PUT /policy/toggle/<id> Admin Only

Enable or disable a policy

Protected Resource Endpoints

GET /resource/data Auth + Policy

Get protected data (authorization enforced)

# Request
curl "http://localhost:5000/resource/data?type=document&department=engineering" \
  -H "Authorization: Bearer YOUR_TOKEN"
GET /resource/documents

Get department documents

GET /resource/reports

Get reports (sensitive reports require elevated access)

Audit Endpoints

GET /audit/logs Auth Required

Get authorization audit logs (paginated)

# Request with filters
curl "http://localhost:5000/audit/logs?page=1&per_page=20&decision=deny&action=write" \
  -H "Authorization: Bearer YOUR_TOKEN"
GET /audit/stats Admin Only

Get authorization statistics

Error Responses

400

Bad Request

Invalid request body or missing required fields

401

Unauthorized

Missing or invalid JWT token

403

Forbidden

Policy evaluation denied access

404

Not Found

Resource not found
